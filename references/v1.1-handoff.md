## Key runtime assumptions

- direct TTS backend: `http://10.40.40.40:9100/v1/audio/speech`
- do not rely on nginx `/tts/...` routes for styled-voice
- Discord cached audio can have misleading extensions; normalize with `ffmpeg` if raw upload fails
- return OGG/Opus to the user for playback compatibility

## v1.1 priorities

### Priority 1 — real usage UX polish
- document `/styled-voice` usage in a concise user-facing form
- add example prompts / style phrasing cheat sheet
- decide how users should specify stronger hesitation / pacing / softness / whisperiness
- possibly add a short README section optimized for Discord usage

### Priority 2 — automatic retry/normalize hardening
- make the skill workflow more robust when raw Discord cached audio fails
- ideal flow:
  1. inspect extracted audio paths
  2. try direct upload when appropriate
  3. if backend fails or media is suspicious, normalize to WAV with `ffmpeg`
  4. retry automatically
  5. always convert final output to OGG/Opus
- improve error surfacing when backend returns non-audio or fails

### Priority 3 — operational stability
- investigate and document nginx/front-door unhealthy `502` state separately from styled-voice
- verify coexistence stability of `vllm` and `voxcpm`
- define minimal operational playbook for startup / memory pressure / restart behavior
